CPS530 Lab 10 – MaterializeCSS Demo

This page is my solution for Problem 2 of CPS530 Lab 10. I used the MaterializeCSS front-end framework to style a simple static page. Below I explain how I used the framework, how I built this page, and the problems I ran into.

1. How I installed / used the framework

I decided to use MaterializeCSS because it is easy to add to a normal HTML file with a CDN link. I did not have to install anything on my computer.

  • I created a new file called index.html.
  • In the <head> section I added the Materialize stylesheet link from the CDN:
    <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/materialize/1.0.0/css/materialize.min.css">
  • I also added the mobile viewport meta tag so the layout looks better on small screens.
  • After that, I could use Materialize classes like container, card and nav-wrapper directly in my HTML to style the page.
2. How I built this page

I kept the page simple and used only a few components of Materialize:

  • I added a navbar at the top using the <nav> tag with the classes nav-wrapper and blue darken-2.
  • I wrapped the main content inside a container to center it and limit the width.
  • I used cards to group each section. Each card has a card-title for the heading and some simple text.
  • For the bullet points, I used Materialize collection lists so they match the overall style.

The page is completely static, but it demonstrates how a framework can improve the look with very little CSS written by me.

3. Difficulties and how I solved them

I had a few small issues while working with the framework:

  • At first the page did not look any different than normal HTML. I realized I had forgotten to include the MaterializeCSS CDN link in the <head> section. After I added the stylesheet link and refreshed the page, the styles appeared correctly.
  • The navbar text was touching the edges of the browser window. I fixed this by wrapping the navbar content inside a container div so it has left and right margins.
  • On mobile, the text felt a bit cramped. I added some extra margin on the main container and a light grey background colour in a small custom CSS block to make it easier to read.

Overall the framework was straightforward to use. Most of the work was just reading the Materialize documentation and applying the correct class names.
